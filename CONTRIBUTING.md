# Contributing

Changes are welcome through pull requests.

1. Do not add private, paid, leaked, or unpublished source material.
2. Add repositories and documentation paths only through a reviewed allowlist change.
3. Run `npm run sync`, `npm run build`, `npm test`, and the skill validator.
4. Preserve source URLs and commit provenance in generated output.

Use a major version for breaking MCP or catalog schema changes, a minor version for new scripts or documentation capabilities, and a patch version for corrections.
